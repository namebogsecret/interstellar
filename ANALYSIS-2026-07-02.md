## 1. Физические ошибки (по убыванию важности)

### 1.1 Drag от гелиоцентрической скорости — БАГ ☑
`js/physics/ship.js:107` — сопротивление считается от `this.v` (система Солнца),
а не от скорости относительно атмосферы. Земля несёт ~30 км/с орбитальной
скорости → корабль, неподвижный относительно Земли в её атмосфере, испытывает
«ветер» 30 км/с ≈ паразитные ~13 м/с² у поверхности. Комментарий «ignore wind»
маскирует не вращение атмосферы (~465 м/с), а орбитальное движение планеты.

**Фикс:** `v_rel = v − v_body` (минимум); в идеале ещё `− ω×r` (соврашение).
`bodyVelocity()` уже есть в `main.js:75` — нужно пробросить в step.

### 1.2 Знак аберрации перевёрнут — БАГ ☑
`js/render/relativisticPass.js:60` — обратное преобразование написано как
`cosθ = (cosθ′ + β)/(1 + β·cosθ′)`, должно быть `(cosθ′ − β)/(1 − β·cosθ′)`.

Числовая проверка: β = 0.5, звезда на траверзе rest-frame (θ = 90°) наблюдается
на θ′ = 60°. Значит пиксель θ′ = 60° должен сэмплировать rest-контент с 90°;
формула кода даёт 36.9°. Итог: передняя полусфера **раздувается** (зум вперёд),
а настоящая аберрация **сжимает** её к центру.

Доплер при этом ВЕРНЫЙ (`D = √(1−β²)/(1−β·cosθ′) ≡ 1/(γ(1−β·cosθ′))`), beaming
честно задокументирован как смягчённый (D^2.2 вместо болометрического D^4).

**Фикс:** флип знака + учесть, что с правильным знаком передние пиксели
сэмплируют углы шире FOV → рендерить источник с расширенным FOV или кубмапой,
иначе клампы по краям на больших β.

### 1.3 Расход топлива по координатному времени — несоответствие ☑
`js/physics/ship.js:88` — `dm = (F/ve)·dt`. Двигатель живёт в системе корабля:
`dm = (F/ve)·dτ = (F/ve)·dt/γ`. На γ = 13 топливо утекает в 13 раз быстрее
положенного, и Δv-бюджет по релятивистскому Циолковскому (HUD) расходится с
фактом. **Фикс — однострочный:** делить на `gammaFromW(this.w)`.

### 1.4 Севший корабль не вращается с планетой ☑
`js/main.js:250-252` — `landedOffset` хранится в инерциальной системе, планета
крутится под кораблём. На Земле стоящий корабль должен нести ~465 м/с на
экваторе. **Фикс:** offset в теле-фиксированной системе (поворачивать через
`spinAngle`), при взлёте добавлять скорость вращения поверхности.

### 1.5 Поперечная тяга при больших γ — оговорить ☑
`dw/dt = a` точна для продольного разгона (гиперболическое движение; верификация
γ≈13 @ 1000g — именно этот случай), но для тяги поперёк скорости ощущаемое
ускорение в γ раз больше заданного. Минимум — оговорка в README; максимум —
честная 4-сила.

## 3. UX

### Ловушки (убрать) ☑
- **Ctrl+W закрывает вкладку**: Ctrl = трим газа вниз (`js/render/controls.js:69`),
  W = тяга. Shift (трим вверх) конфликтует с Shift+Tab (цикл целей назад).
  → перенести трим на `-`/`=` или `[`/`]`.
- **Ловушка новичка в аркаде**: первое W ставит throttle=1 = 1000g
  (`controls.js:85` + `ship.js:22`) — орбита уничтожена за полсекунды.
  → лог-шкала цифр (1=1g … 9=1000g), дефолт 1–2g.
- **Нет паузы**: минимум варпа 1×. → клавиша паузы (warp 0).

### Навигационный слой (самый большой выигрыш) ☑

  HUD — сейчас не понять, стабильна ли орбита, пока не упадёшь. Всё для коники
  по вектору состояния уже есть (`orbitEllipse` рисует эллипсы).
- **Клавиша «циркуляризовать»** рядом с X: X возле планеты = «начать падать»
  (правильная физика, но новичок ждёт «стабилизируй»). ~5 строк.
- **Честный ETA**: сейчас `d/v` (`js/render/hud.js:35`) — бесполезно для
  релятивистского перелёта. → брахистохрона (разгон-переворот-торможение)
