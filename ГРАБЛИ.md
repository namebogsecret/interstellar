## 2026-07-03 · render/relativisticPass+scene · регрессия FOV при отключаемом пассе
Симптом: отключение рел-оптики (клавиша C или восстановленный `iss_relfx=false`) резко «отзумливало» сцену — весь мир показывался в 90° вместо 60°.
Корень: базовый `RenderPass` стали рендерить через `sourceCamera` (SOURCE_FOV=90°, добавлен ради запаса на аберрацию), а обратно до 60° кадр кропит ТОЛЬКО шейдер рел-пасса; когда `relativistic.enabled=false`, EffectComposer пропускает пасс → некропнутый 90° растягивается на экран. Правило: любой always-on инвариант (кроп кадра) нельзя вешать на отключаемый пасс.
Регресс-тест: Playwright-smoke жмёт C и сверяет кадр on/off (scratchpad relfx_on/off.png); фикс — свап `composer.renderPass.camera = sim.relFx ? sourceCamera : camera` в цикле рендера + startup-apply.

## 2026-07-03 · data/bodies · планета мимо цикла инициализации (guard съел один элемент)
Симптом: после перехода на реальные J2000-долготы Нептун оказался единственной планетой не на своём месте (остался с выдуманным `M0`).
Корень: цикл `for (b of BODIES) if (typeof b.L0==='number') b.M0 = j2000MeanAnomaly(...)` пропускает тела без `L0`; Нептуну забыли добавить `L0`, и guard молча его исключил (не краш, не NaN — просто фиктивная фаза). Правило: при частичной миграции по guard'у — проверить, что КАЖДЫЙ элемент, который должен попасть, реально имеет ключ.
Регресс-тест: probe `BODIES.find(b=>b.name==='Neptune').M0` ≠ 0 и ≠ старого 2.9.

## 2026-07-03 · i18n/hud · знак съедает выбор единиц измерения
Симптом: скорость сближения при удалении (отрицательная) ≥1 км/с печаталась как `-100000000.0 m/s` вместо `-33.36 %c`.
Корень: `fmtSpeed(v)` выбирал ветку единиц по ЗНАКОВОМУ `v` (`if (v<1e3)…`), а не по `Math.abs(v)`, как `fmtDist/fmtTime`; прежние вызовы всегда слали ≥0, новая строка HUD — знаковую. Правило: форматтеры величин ветвятся по модулю, знак — только в выводе.

## 2026-07-03 · physics/main · алиасинг модульных scratch-векторов (главные грабли репо)
Симптом (потенциальный, пойман на дизайне): переиспользуемые `_tmp/_rel/_dir/_refVel` и т.п. — если один и тот же scratch прочитан после того, как его перезаписал другой участок того же кадра, получаются тихие числовые ошибки (дрейф орбиты, сломанный warp-cap).
Корень/правило: (1) `refBodyVel`, приходящий в `ship.step`, — это общий `_refVel` из main.js; step обязан КОПИРОВАТЬ его, не мутировать. (2) nav-объект для HUD и промежуточные векторы посадки должны использовать ВЫДЕЛЕННЫЙ scratch (`_navRefPos`, `_landOff`…), не `_tmp/_rel/_dir`, которые клоберятся ниже по кадру (floating-origin loop, landed-блок). (3) `landedOffset` при повращении каждый кадр — клонировать перед `applyAxisAngle` (мутирует in-place).

## 2026-07-03 · render/relativisticPass · early-out при широком источнике
Симптом (пойман на дизайне): с широким (90°) источником ветка early-out `if(eff<1e-4)` не должна сэмплить сырой `vUv` — иначе в покое вид «отзумлен».
Корень/правило: при v→0 возвращать `texture(tDiffuse, uvRestDir)` (направление дисплей-FOV, репроецированное в широкий источник), а не `vUv`. GLSL-формула аберрации `(cp-beta)/(1.0-beta*cp)` обязана быть байт-в-байт как `aberratedCos()` в relativity.js — при расхождении шейдер молча разойдётся с тестируемым мирором.
